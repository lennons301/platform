Chapter 2

Interlude — Motivation

Wanted to dispatch development tasks from my phone.

Tools like Lovable exist but you give up control — no git history, no real containers, no custom tooling. Wanted the convenience with full ownership.

So I built Interlude

A self-hosted platform: describe a task → AI agent works in a Docker container → streams output to a chat UI → pushes a branch → you review the PR.

The key motivation: I want to be on my phone, describe what I want, and have an agent do the work. Like having a junior dev you can message anytime.

Chapter 3

Scaling Problems

Started converting Premier League Survivor Picks (a Lovable-deployed project) and hit walls fast.

Supabase limits
Only 2 free projects. Can't sustain environment separation (dev/stg/prd) across multiple products.
Environment inconsistency
Local dev in parts, production database in parts. No clear boundary between environments.
Secrets management
.env.local files on one laptop. Doesn't work when agents run on a VPS. Single point of failure.
Version drift
Already losing track of what version of what framework is in which project and why.
This is the turning point. One app is fine. Two apps is fine. Three apps with agents developing them remotely — everything falls apart without structure.
Chapter 4

The Platform Repo

A single repo that defines what the estate should look like — consumed by both humans and agents.

platform/
├── standards/     5 principles (environments, secrets, local-dev, docs, testing)
├── choices/       6 decision areas with matrices (db, auth, hosting, orm, secrets, ui)
├── versions/      Target runtimes (Node 22, Next.js 16, React 19, TS 5.7)
├── products/      Registry: each project's choices, versions, divergences
├── checks/        Shell scripts that validate conformity
├── templates/     Starters for new projects
└── .github/       CI runs checks on push, creates issues for gaps
Simple structure. All YAML and markdown. The key insight: this is consumed by agents too — it's mounted into every Interlude container so agents know the rules.
Chapter 4

Standards

Five tool-agnostic rules every project must follow — regardless of which database or framework is chosen.

Environments

Three isolated environments: dev (local, disposable), stg (remote, safe to reset), prd (production). Separate secrets per environment.

Secrets

Single remote provider as source of truth. Never committed, never hardcoded. .env.local is a cache, not canonical.

Local Development

All projects run against local Docker services. If your laptop is offline, development still works. No dependency on remote databases.

Documentation

Every repo has a CLAUDE.md — the primary technical reference for both humans and AI agents. Updated at the end of each phase.

Testing

Automated tests running against local services. CI runs them on every PR.

These are the non-negotiables. They're tool-agnostic — swap Neon for Supabase and the standards still hold. The CLAUDE.md standard is the AI-specific one — every agent that touches a repo gets context automatically.
Chapter 4

Choices

Decision matrices with a preferred default for each area. Projects can diverge with a documented reason.

AreaDefaultWhy
DatabaseNeon (serverless Postgres)10 free projects, local Docker for dev
AuthBetter AuthData in your DB, pairs with Drizzle
ORMDrizzleType-safe, works locally and remotely
SecretsDopplerCentral store, agents can access it too
HostingVercel Hobby (free)Zero-config deploys
UIshadcn/ui + TailwindSource-owned components

Total fixed infrastructure cost: ~€4.50/mo (just the Interlude VPS). Everything else is free tier.

Each choice has a full decision matrix with alternatives. The defaults are optimised for free-tier-first — this can't cost more than just paying for hosted services.

Chapter 4

The Intended Loop

Platform defines standards + choices
↓
CI runs conformity checks across all repos
↓
Gaps auto-create GitHub Issues
↓
Agents fix via Interlude (platform context mounted)
↓
PR merged → checks re-run → estate converges

Today: the loop is partly manual. Next: agents pick up conformity issues automatically.

This is where it all connects. Platform is the brain, Interlude is the hands. Agents inherit the platform context so they produce conformant code by default. The goal is to close this loop fully — gap detected, agent fixes it, PR filed, no human needed for routine fixes.
